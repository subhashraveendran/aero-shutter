# AeroShutter app icon & splash assets

## Concept

A branded "darkroom instrument" mark: a multi-blade **camera aperture / iris**
rendered as a bold amber disc, its blades meeting at a small central pinhole
opening, glowing on a near-black darkroom background.

- **Foreground:** 6-blade aperture in a warm **amber → deep-amber** gradient
  (`#ffd178 → #ffab2e → #a8490a`). Dark blade seams (`#0a0a0b`) radiate from a
  central opening ringed with a thin warm highlight (`#ffe0a0`), giving depth.
- **Background:** subtle radial darkroom field, `#14110d` center → `#0a0a0b`
  edges, with a soft amber center glow.
- The silhouette is a simple, bold filled disc, so it stays legible at small
  launcher sizes. No thin lines, no text.

## Source files (`resources/`)

Hand-authored vector art, generated by `resources/generate-source.mjs` (the
aperture geometry is computed, not drawn by hand):

| File                       | Size       | Purpose                                             |
| -------------------------- | ---------- | --------------------------------------------------- |
| `icon.svg`                 | 1024       | Vector master (full-bleed aperture on dark bg)      |
| `icon.png`                 | 1024x1024  | Rasterized master icon, full-bleed                  |
| `icon-foreground.png`      | 1024x1024  | Adaptive-icon foreground (aperture, safe padding)   |
| `icon-background.png`      | 1024x1024  | Adaptive-icon background (darkroom field)           |
| `splash.png`               | 2732x2732  | Splash — centered small aperture on darkroom bg     |
| `splash-dark.png`          | 2732x2732  | Dark-mode splash (same darkroom art)                |

### How the SVG is rasterized (no ImageMagick)

`resources/generate-source.mjs` uses the **`sharp`** library (a transitive
dependency of `@capacitor/assets`) to render the SVG strings to PNG at a high
density (384 dpi) and resize to the target dimensions. It writes all six PNGs
plus the SVG master. No native image tooling is required.

## Generated platform assets

Produced by `@capacitor/assets` (`capacitor-assets generate`):

**Android** — `android/app/src/main/res/`
- `mipmap-{mdpi,hdpi,xhdpi,xxhdpi,xxxhdpi}/ic_launcher.png` (48/72/96/144/192)
- `.../ic_launcher_round.png` (same densities)
- `.../ic_launcher_foreground.png` and `ic_launcher_background.png` (108→432)
- `mipmap-anydpi-v26/ic_launcher.xml` + `ic_launcher_round.xml` (adaptive icon)
- `drawable-*/splash.png` including `-night-` (dark) and `-land-`/`-port-` variants

**iOS** — `ios/App/App/Assets.xcassets/`
- `AppIcon.appiconset/AppIcon-512@2x.png` (1024x1024) + `Contents.json`
- `Splash.imageset/` — `Default@{1x,2x,3x}~universal~anyany.png` and matching
  `-dark.png` variants (2732x2732) + `Contents.json`

The regenerated icons are new art (different bytes than the stock Capacitor
launcher — verified by checksum before/after).

## Launch screen colors

`capacitor.config.ts` uses the darkroom color `#0a0a0b` for `backgroundColor`,
the `SplashScreen` `backgroundColor`, and the `StatusBar` `backgroundColor`
(style `DARK`), so the native launch screen matches the icon background.

## Regenerating

```
npm run assets
```

This runs `assets:source` (rebuilds the `resources/` PNGs from the SVG via
`sharp`) then `capacitor-assets generate` with the darkroom background colors:

```
--iconBackgroundColor '#0a0a0b' --iconBackgroundColorDark '#0a0a0b'
--splashBackgroundColor '#0a0a0b' --splashBackgroundColorDark '#0a0a0b'
```

Note: this project has no PWA `www/manifest.json`, so `capacitor-assets` prints
a harmless `ENOENT .../www/manifest.json` at the very end when it reaches the
PWA step. The iOS and Android assets are fully written before that point and are
unaffected. To target a platform only, the native output is what matters here.
